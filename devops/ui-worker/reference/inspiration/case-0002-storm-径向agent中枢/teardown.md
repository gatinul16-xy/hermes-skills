## 一句话为什么好看
用"中央发光内核 + 环绕的 Agent 节点"把"多智能体协作"具象成一个能量场，比任何列表式布局都更有记忆点和"活着"的感觉。

## 视觉拆解
- **配色**：深空蓝黑底；中央核心青色辉光；每个 Agent 一个**身份色**——深(青 #4cd2c2 系)、灵(珊瑚红)、你(紫)。强调色不是装饰，而是承担"身份区分"功能。背景 < #0a0e14，卡片半透明深蓝。
- **字体/排版**：无衬线为主，干净；分区标题(议题/知识库/AGENTS/SESSION OUTPUT)用小字大写或弱色做"分区眉标"；核心区 `STORM CORE` + 副标 `Coordination · Analysis · Promotion` 居中。
- **布局/栅格**：经典三栏(左输入与配置 / 中可视化舞台 / 右输出)，顶部阶段步进条 + 右上 Tab。中央用**径向同心圆轨道**承载 Agent，核心居中、节点沿轨道分布。
- **层次手法**：靠"发光强弱 + 大小 + 位置(中心>轨道>边栏)"建立主次；核心最亮最大。
- **圆角/材质**：统一中等圆角；卡片轻玻璃质感；同心轨道用虚线，弱化为背景结构。
- **动效(推测)**：核心波形脉动、辉光呼吸、节点待机微动——传达"就绪/在场"。
- **最妙细节**：① Agent 用单字头像 + 身份色，识别极快；② 阶段条把抽象流程(议题准备→自由讨论→交叉反驳→共识整理→投票决策)显性化；③ 核心副标三词点出系统职责。

## 可复用要点（重点）
- **"中心内核 + 环绕节点"的径向布局**：表达"一个协调中枢 + 多个参与体"的关系(多 Agent、多模型、多服务、星系/拓扑)时，比流程图/列表更有记忆点。
- **身份色系统**：给每个参与体分配一个固定强调色 + 单字头像，跨界面一致，识别成本极低。
- **抽象流程显性化**：把多阶段流程做成顶部步进条，配状态高亮。
- **同心虚线轨道当背景结构**：弱化为氛围，不抢主体；核心用辉光+波形表"在线/思考"。
- **三栏:输入配置 | 可视化舞台 | 实时输出** —— 适合任何"配置→运行→产出"的 Agent/工具类应用。

## 适用场景
多智能体/多模型协作、编排器、AI 工具控制台、需要表达"中枢—参与体"关系的深色科技界面。

## 待改进（别学的部分）
- 中央留白过大→待机态需加引导/微动，避免空旷。
- 次要文字(STANDBY/空态/placeholder)对比过低→提到 ≥WCAG AA。
- 右侧 output 空态太冷清→给骨架/示意。
