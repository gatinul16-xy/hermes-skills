# 设计经验沉淀（lessons）

每次交付后，若出现一个新的、好用的可复用模式，追加到这里（一行一条，带场景）。
与 INDEX 的区别：INDEX 是"看到的好作品拆解"，这里是"我自己做的过程中验证有效的做法"。

- 深色 B 端：单强调色面积 <10%、只给 CTA/关键数字/状态，最省力地拉专业感。
- 落地页 Hero：非对称 + 一张"会动的产品演示卡"做钩子，比纯文案标题留人。
- 多图展示：用统一尺寸卡片框 + object-fit（横图 cover、竖图 contain），不同尺寸截图也叠得齐（见 remotion Stack）。
- 避免死静态/视觉平庸：元素入场要铺开时间、配持续微动，不要一次性全淡入。
- Bento/网格布局：每一行的卡片跨度必须加起来填满总列数（如 12 列：4+5+3、5+7…），否则 auto-flow 会留下尴尬空格。布局完成后逐行核对列和。
- 临摹参考图时要复刻"全部关键区块"，别为了精简删掉某块内容——固定高度容器(如手机屏)里删了内容会留下大片空白。核对：参考有几个 section，产出就得有几个，最后一屏内容要填到底或自然收尾。
