# 设计评审打分表（Review Rubric）

给一张设计稿(自己产出的或别人的)按维度打分 + 给必改项。用于交付前自评、或评他人稿。
每项 0–5 分(0 缺失 / 3 合格 / 5 顶尖),乘权重得加权分;<3 的项必须列进"必改"。

| 维度 | 权重 | 看什么 | 0–5 |
|---|---|---|---|
| 首屏 / 焦点 | 3 | 3 秒内有没有明确视觉重心、看不看得懂;一屏一个重心 | |
| 层次 | 3 | 主次靠字重/对比/留白拉开,而非堆边框分割线 | |
| 配色 | 2 | ≤1 强调色、面积克制、无 AI 紫蓝/纯黑/过饱和、阴影染色 | |
| 排版 | 2 | 字体有性格(非默认 Inter)、字号阶梯清晰、行宽/行高合理、仪表盘不用衬线 | |
| 间距 / 对齐 | 2 | 8pt 标度、对齐到栅格、留白有呼吸、无歪斜 | |
| 一致性 | 2 | 圆角/图标线宽/色板/组件风格统一;符合品牌记忆 | |
| 动效 / 微交互 | 2 | 全状态(hover/focus/active/disabled/空/错)、缓动自然、只动 transform/opacity、无死静态 | |
| 无障碍 | 1 | 对比 ≥WCAG AA、焦点态可见、不单靠颜色传意 | |
| 细节收口 | 2 | 圆角统一、对齐精确、无半成品/占位/假数据/emoji | |
| 原创 / 记忆点 | 1 | 不套通用模板、有记忆点、非抄袭 | |

## 判定
- 加权满分 = 100。**≥85 顶尖 / 70–84 良好 / 55–69 及格 / <55 需返工。**
- 任一维度 0 分 → 直接需返工。
- 输出格式:逐维度分数 + 一句理由 → 总分与判定 → **必改项清单(<3 分的维度,给具体改法)** → 加分建议。

## 用法
- 交付前 ui-worker 用它自评,不达 70 先改再交。
- 评他人/参考稿时,顺手把"顶尖项"的可复用手法记进 `inspiration/`。
